# 페이지 제작 아이디어

## 모든 데이터를 동적 데이터 관점으로 접근한다
    - 이점
        - 관리자 모드를 통해 내용 변경이 용이한 구조
        - 언어 변경시 페이지의 내용 변경이 용이한 구조
    - 방법
        - 기획서와 디자인 파일을 받는다.
        - 기획서와 디자인 파일을 기반으로 페이지를 구성하는 데이터를 json파일로 작성한다.
            - 이때 데이터의 의미에 따르는 규칙(예: FXTSX-JSON)에 의거하여 작성한다.
            - json 파일의 key값을 기반으로 i18n라이브러리를 이용하여 HTML5 코드를 작성한다.
            - 작성된 json은 db로 저장한다.
        - i18n을 셋팅한다.
            - 사용자가 특정 국가에서 페이지를 로드하면 서버에서 json 파일을 chat-gpt-api를 이용하여 [페이지의 한글 데이터].json 를 [해당국가의 언어 데이터].json 으로 번역하여 db에 저장한다.
                - 초기 로드는 번역이 진행됨으로 느릴 수 있음
                - 초기 로드 이후 에는 db에서 페이지 데이터를 가지고 오기에 모든 사용자가 정상적인 서비스를 경험
                - 서비스 관리 업체 내부에서 초기 페이지 로드를 진행
                - 해당 부분은 관리자 모드에서 진행 가능
        - 관리자 모드를 셋팅한다.
            - 페이지의 모든 데이터는 관리자에 의해 동적으로 추가 삭제가 가능하다.
            - 수정후 db내 모든 언어별 데이터 파일과 싱크를 맞춘다.
